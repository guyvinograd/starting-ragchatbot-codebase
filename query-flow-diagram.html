RAG Chatbot — Query Flow

1

User Input

frontend/script.js : sendMessage()

User types a question and presses Enter or clicks Send. Input is disabled and a loading animation appears.

POST /api/query → { query, session_id }
2

FastAPI Route

backend/app.py : query_documents()

Receives QueryRequest. Creates a session if session_id is null, then delegates to the RAG system.

rag_system.query(query, session_id)
3

RAG Orchestrator

backend/rag_system.py : query()

Wraps query in a prompt, fetches conversation history from SessionManager, and calls the AI generator with tool definitions.

ai_generator.generate_response(prompt, history, tools)
4

Claude API Call

backend/ai_generator.py : generate_response()

Sends system prompt + user message + tool definitions to Anthropic API with tool_choice: auto. Claude decides whether to search or answer directly.

Claude's stop_reason?

Direct Answer

General knowledge question — Claude responds with text immediately. Returns response.content[0].text.

Tool Use → Step 5

Course-specific question — Claude requests search_course_content tool call with query, course_name, lesson_number.

5

Tool Execution

backend/search_tools.py : CourseSearchTool.execute()

Called by ToolManager. Delegates to the vector store's search interface with optional course and lesson filters.

vector_store.search(query, course_name, lesson_number)
6

Vector Search (ChromaDB)

backend/vector_store.py : search()

Resolves fuzzy course names via semantic search on course_catalog. Builds a where-filter, then queries course_content collection. Returns top 5 matching chunks.

formatted results → 2nd Anthropic API call → final answer
7

Claude Synthesizes Answer

backend/ai_generator.py : _handle_tool_execution()

Tool results are appended to the message history. A second API call (without tools) lets Claude synthesize a final answer from the search results.

{ answer, sources[], session_id } → JSON response
8

Render Response

frontend/script.js : addMessage()

Answer is parsed as Markdown via marked.js and displayed. Sources appear in a collapsible <details> element. Session ID is stored for follow-up queries.
